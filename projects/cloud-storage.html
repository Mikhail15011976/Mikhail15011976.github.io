Облачное хранилище (Fullstack: Django + React)

Fullstack веб-приложение для безопасного хранения файлов с production-развертыванием на VPS (Nginx + Gunicorn + PostgreSQL)

Python 3.11 Django 5 Django REST PostgreSQL React 19 JWT Nginx Gunicorn Ubuntu Server Systemd SSL/TLS

Основные функции

  • JWT & Session Authentication — Гибкая система аутентификации и авторизации
  • RESTful API на Django REST Framework — Полноценное API для всех операций
  • Advanced File Handling — Загрузка, скачивание, категоризация, контроль типов и размеров
  • PostgreSQL с оптимизацией — Надежное хранение данных с индексацией для быстрого поиска
  • Modern UI/UX с Material-UI — Чистый, адаптивный интерфейс на React 19
  • Drag & Drop Upload — Удобная загрузка файлов перетаскиванием
  • Real-time Updates — WebSocket уведомления о статусе загрузки
  • Production-развертывание — VPS с Nginx, Gunicorn, SSL на порту 8443

Архитектура системы

Backend (Django REST)

  • Django 5.2.1 — основной фреймворк
  • Django REST Framework — построение API
  • Simple JWT — аутентификация по токенам
  • PostgreSQL 12+ — production база данных
  • Gunicorn — WSGI сервер
  • Swagger/Redoc — документация API

Frontend (React SPA)

  • React 19.1.0 — современный интерфейс
  • Material-UI (MUI) — компонентная библиотека
  • React Router DOM — навигация SPA
  • Axios — HTTP клиент для API
  • WebSocket — real-time уведомления
  • Context API — управление состоянием

DevOps & Infrastructure

  • Ubuntu Server 20.04+ — операционная система
  • Nginx 1.18+ — reverse proxy и SSL терминация
  • Systemd — управление сервисами
  • SSL/TLS на порту 8443 — защищенное соединение
  • UFW брандмауэр — настройка безопасности
  • Автоматический деплой — скрипты обновления
Системная архитектура (production):

Client → Nginx:8443 (SSL) → Gunicorn → Django → PostgreSQL
│
├── Static Files (React Build)
└── Media Files (Uploads)

Решенные задачи и принятые решения

Задача 1: Production-развертывание с SSL на порту 8443

Проблема: Требовалось развернуть fullstack приложение с отдельной обработкой статики (React), медиафайлов и API, используя безопасное SSL соединение на нестандартном порту.

Решение:

  • Nginx конфигурация: Настройка как reverse proxy с SSL терминацией на порту 8443
  • Разделение статики: Nginx отдает собранные React файлы и медиафайлы, проксирует API запросы на Gunicorn через Unix socket
  • Systemd сервис: Создание production-ready systemd сервиса для Django с настройками Restart=always, RestartSec=10
  • SSL сертификаты: Генерация самоподписанных сертификатов для шифрования трафика

Задача 2: Оптимизация работы с файлами и базой данных

Проблема: Эффективное хранение метаданных файлов, быстрый поиск и скачивание, защита от переполнения диска.

Решение:

  • PostgreSQL с индексацией: Использование производственной БД с индексами на полях поиска (имя, тип, дата)
  • Квоты пользователей: Реализация системы лимитов дискового пространства на уровне модели пользователя
  • Валидация файлов: Проверка MIME-типов, размера, расширений перед сохранением
  • Эффективное хранение: Метаданные в БД, файлы в файловой системе с хэшированными путями

Задача 3: Безопасность и авторизация

Проблема: Защита API от несанкционированного доступа, безопасная обработка загружаемых файлов, защита от распространенных веб-атак.

Решение:

  • JWT аутентификация: Реализация через django-rest-framework-simplejwt с refresh токенами
  • Защита от XSS/CSRF: Использование встроенных механизмов Django и Django REST Framework
  • Безопасность файлов: Проверка загружаемых файлов на вирусы (в идеале), валидация расширений и MIME-типов
  • Защитные заголовки Nginx: Настройка X-Frame-Options, X-Content-Type-Options, HSTS

Задача 4: Автоматизация деплоя и мониторинга

Проблема: Ручное обновление fullstack приложения (backend + frontend) сложно и подвержено ошибкам.

Решение:

  • Скрипт автоматического деплоя: Создание deploy-and-restart.sh, который выполняет полный цикл обновления
  • Мониторинг через systemd: Использование journalctl -u cloud-storage -f для просмотра логов
  • Резервное копирование: Реализация скриптов для бекапа базы данных и медиафайлов
  • Проверка здоровья: Настройка проверок доступности API и статических файлов
