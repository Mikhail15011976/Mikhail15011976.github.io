Казанский храм - Официальный сайт Production - Онлайн

Профессиональный информационный портал на Flask с единой архитектурой страниц, адаптивным дизайном и полным CI/CD циклом production-развертывания

Python 3.11 Flask 3.1 Jinja2 HTML5/CSS3 JavaScript Nginx Gunicorn Ubuntu Server Systemd GitHub Actions CI/CD

Основные функции и архитектурные решения

  • Единая архитектура страниц — Все информационные разделы в одном шаблоне с динамической загрузкой через Jinja2
  • Flask микрофреймворк — Легковесное и быстрое приложение на Python 3.11 с минимальной зависимостью
  • Интерактивные фотогалереи — Полноэкранный просмотр с навигацией клавишами и тач-жестами на чистом JavaScript
  • 100% адаптивный дизайн — 3 брейкпоинта (mobile, tablet, desktop) с оптимизацией изображений
  • Полный CI/CD цикл — Автоматический деплой через GitHub Actions при каждом коммите в main ветку
  • Production-ready инфраструктура — VPS с Nginx как reverse proxy, Gunicorn WSGI сервер, Systemd управление
  • Безопасная конфигурация — Изолированное окружение, SSH-ключи для доступа, настройка брандмауэра
  • Система пожертвований — Банковские реквизиты и информация о срочных проектах храма

Системная архитектура (production):

Client → Nginx (порт 80) → Gunicorn WSGI → Flask App
│
├── Static Files (HTML, CSS, JavaScript)
└── Media Files (Фотографии храма)

CI/CD Pipeline:
Push в main → GitHub Actions → SSH на сервер → Обновление кода → Перезапуск сервиса

Ключевые технические решения и решенные задачи

Задача 1: Единая архитектура страниц на Flask + Jinja2

Проблема: Необходимо было создать 8+ информационных разделов с единым дизайном без дублирования кода.

Решение:

  • Единый шаблон about.html: Все разделы используют один шаблон с динамическим контентом
  • Jinja2 условные блоки: Отображение контента в зависимости от текущего раздела
  • Flask маршрутизация: Простые эндпоинты для каждого раздела (/about, /clergy, /schedule и т.д.)
  • Результат: Легкое добавление новых разделов и единый код для всего сайта

Задача 2: Создание 100% адаптивного дизайна

Проблема: Сайт должен одинаково хорошо отображаться на всех устройствах от смартфонов до десктопов.

Решение:

  • Mobile-first подход: Верстка начиналась с мобильной версии
  • 3 четких брейкпоинта: Отдельные стили для мобильных, планшетов и десктопов
  • Flexbox и CSS Grid: Современные технологии для гибкой верстки
  • Оптимизация изображений: Автоматическое сжатие и адаптивные теги picture/source

Задача 3: Production-развертывание с полным CI/CD циклом

Проблема: Ручное обновление кода на сервере занимало 15-20 минут и могло привести к ошибкам.

Решение:

  • GitHub Actions workflow: Автоматический деплой при каждом коммите в main
  • Безопасное SSH-подключение: Использование SSH-ключей через GitHub Secrets
  • Systemd сервис: Настройка автозапуска, мониторинга и автоматического перезапуска
  • Архитектура Nginx + Gunicorn: Nginx для статики и SSL, Gunicorn как WSGI сервер
  • Результат: Время деплоя сократилось до 2-3 минут с полной автоматизацией

Задача 4: Обеспечение безопасности и отказоустойчивости

Проблема: Требовалась стабильная работа сайта 24/7 с защитой от возможных атак.

Решение:

  • Выделенный VPS: Изолированное окружение на Ubuntu Server
  • Настройка UFW брандмауэра: Разрешение только необходимых портов (SSH, HTTP, HTTPS)
  • Systemd настройки безопасности: Restart=always, RestartSec=10, PrivateTmp=true
  • Резервное копирование: Автоматическое создание бэкапов через workflow
  • Мониторинг логов: Использование journalctl для отслеживания ошибок
