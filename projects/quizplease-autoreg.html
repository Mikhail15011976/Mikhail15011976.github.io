QuizPlease Autoreg - Парсер и Telegram-бот

Автоматическая система мониторинга расписания игр на quizplease.ru с отправкой уведомлений в Telegram и полным CI/CD циклом

Python 3.11 BeautifulSoup4 Requests Telegram Bot API GitHub Actions Systemd Ubuntu Server JSON CI/CD

Возможности системы

  • Автоматический мониторинг — Парсинг расписания игр с автоматической проверкой изменений
  • Telegram уведомления — Автоматические уведомления о новых играх и изменении статуса
  • CI/CD пайплайн — Автоматический деплой через GitHub Actions при каждом коммите в main
  • Отказоустойчивость — Работает как systemd-сервис 24/7 с автоперезапуском
  • Полная информация — Детализированные сообщения с датой, временем, местом, ценой и статусом
  • Хранение данных — JSON-файлы для хранения текущего расписания и истории изменений
  • Логирование — Комплексное логирование в файлы и systemd journal
  • Конфигурация — Простая настройка через конфигурационный файл

CI/CD Pipeline (GitHub Actions)

  • Trigger: Push в main ветку на GitHub
  • Подготовка: Установка Python и зависимостей
  • Развертывание: SSH подключение к серверу
  • Обновление: Обновление кода из репозитория
  • Установка: Установка зависимостей Python
  • Перезапуск: Перезапуск systemd сервиса
  • Уведомление: Отправка статуса деплоя

Ключевые компоненты

  • extract_classic_games.py — Основной модуль парсера
  • telegram_notifier.py — Telegram бот для уведомлений
  • get_chat_id.py — Утилита для получения Chat ID

  • quizplease-autoreg.service — Конфигурация Systemd сервиса

Решенные задачи и технические решения

Задача 1: Надежный парсинг сайта с динамической структурой

Проблема: Сайт quizplease.ru имеет сложную HTML-структуру, которая может изменяться.

Решение:

  • Многоуровневое извлечение данных: Несколько методов поиска для каждого поля
  • Резервные селекторы: Альтернативные CSS-селекторы на случай изменений структуры
  • Валидация данных: Проверка корректности данных перед сохранением
  • Хэширование: Создание уникальных идентификаторов игр для отслеживания изменений

Задача 2: Обеспечение бесперебойной работы 24/7

Проблема: Система должна работать надежно и восстанавливаться после сбоев.

Решение:

  • Systemd сервис: Production-ready конфигурация с автоперезапуском
  • Комплексное логирование: Логирование в файлы и systemd journal
  • Обработка ошибок: Грейсфул деградация при сбоях сети или сервера
  • Таймауты: Настройка таймаутов HTTP-запросов

Задача 3: Автоматизация деплоя и обновлений

Проблема: Ручное обновление кода на сервере было трудоемким и подверженным ошибкам.

Решение: CI/CD пайплайн на GitHub Actions

  • Автоматический триггер: Деплой при пуше в main ветку
  • Безопасное подключение: SSH-ключи через GitHub Secrets
  • Полный цикл: Обновление кода → установка зависимостей → перезапуск сервиса
  • Уведомления: Отправка статуса деплоя

Задача 4: Интеллектуальные уведомления без спама

Проблема: Избегать спама, но информировать о важных изменениях.

Решение:

  • Только изменения: Уведомления только при новых играх или изменении статуса
  • Полная информация: Детализированные сообщения по каждой игре
  • Форматирование: Читабельные сообщения с эмодзи и Markdown
  • Уведомления об ошибках: Сообщения о проблемах в работе парсера

Задача 5: Безопасность и конфигурация

Проблема: Безопасное хранение чувствительных данных (токены, Chat ID).

Решение:

  • Шаблон конфигурации: config.example.py для безопасного клонирования
  • Отдельный файл: config.py не отслеживается в Git
  • Встроенная утилита: get_chat_id.py для получения Chat ID
  • Изоляция: Сервис работает под отдельным пользователем Linux

Быстрый старт и использование

Установка

# Клонируйте репозиторий git clone https://github.com/Mikhail15011976/quizplease-autoreg.git cd quizplease-autoreg # Установите зависимости pip install -r requirements.txt # Настройте конфигурацию cp src/config.example.py src/config.py # Отредактируйте src/config.py, добавив ваш токен # Получение Chat ID python src/get_chat_id.py

Запуск

# Локальный запуск (для тестирования) python src/extract_classic_games.py # Конфигурация systemd сервиса sudo nano /etc/systemd/system/quizplease-autoreg.service # Включение и запуск сервиса sudo systemctl daemon-reload sudo systemctl enable quizplease-autoreg sudo systemctl start quizplease-autoreg

Управление сервисом

# Проверить статус sudo systemctl status quizplease-autoreg # Просмотреть логи sudo journalctl -u quizplease-autoreg -f # Перезапустить сервис sudo systemctl restart quizplease-autoreg # Остановить сервис sudo systemctl stop quizplease-autoreg
